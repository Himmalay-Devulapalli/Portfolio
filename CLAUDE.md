# Response preferences

Default behavior:

- Be concise.
- Do not explain obvious code.
- DO not generate code or make modifications to file unless specifically asked.
- Do not teach concepts unless explicitly asked.
- Build incrementally.
- Ask questions only if absolutely necessary.
- Keep explanations on point and short, but clear.
- Prioritize implementation over discussion.
- Do not repeat previously explained information.
- DO not use claude previews unless asked.
- Do not provide tutorials unless explicitly requested.


# Coding rules

- Functional components only
- Components under 200 lines only when possible
- Reuse existing components
- Do not create unnecessary files

# Output rules

When asked to build features:
1. Briefly explain the plan (1-2 bullets max, only if required).
2. Implement changes
3. Respond the success/failure status with clear explanations(2 bullets if the task is successfully done, 10 bullets max if failed with on point reasoning.)
3. Stop
